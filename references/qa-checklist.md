# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净纯白。
- 有呆头黑猫。
- 黑猫承担核心动作，不只是装饰。
- 黑猫占比克制：默认约 8%-18%，最多约 25%，不能抢走结构主体。
- 黑猫有明确动作：扒拉、轻微缠线、叼纸、拍按钮、坐节点、钻抽屉、尾巴勾线等。
- 角色识别点清楚：黑色身体、大白眼、小黑瞳、三角耳、少量乱线胡须、呆滞表情。
- 整体画面接近 Ian 原版小黑项目：极简、纯白、手绘、留白、克制、怪诞但清爽。
- 没有复刻参考图近距离头像构图，而是为当前文章生成了新隐喻。
- 简洁清爽，主体结构不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 橙色只用于主路径或箭头。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“常见坑 / Workflow / 系统架构图 / 路线图”等标题。
- 黑猫太大，第一眼只看到猫，看不清结构。
- 黑猫像真实猫、宠物头像、表情包贴纸或甜美儿童插画。
- 黑猫只是趴着或站着，没有参与动作。
- 黑猫仍是近距离正面大头，手脚、尾巴和身体没有参与表达。
- 画面像 PPT、课程课件、正式流程图。
- 画面太粗糙、太脏、太水彩、太满，偏离 Ian 原版清爽手绘风。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点或复古质感。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 角色失去固定识别点，尤其是大白眼、三角耳和少量胡须。
- 和参考图过于相似，只是黑猫头像换表情。

## 迭代方法

- 黑猫太大：缩到画布 8%-18%，把主体还给低科技隐喻。
- 太普通：让黑猫成为小动作主体，加入一个猫式误操作隐喻。
- 动作太单一：改成轻微缠线、扒拉纸卡、叼标签、坐节点、尾巴勾路径。
- 太乱：删节点，只保留一个动作和 3-5 个短标注。
- 太水彩：去掉背景纹理和大面积毛边，只保留黑猫身体一点粗糙边缘。
- 太可爱：强调 blank confused expression, deadpan, not cute, not pet sticker。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘低科技场景。
- 太像参考图：保留角色特征，换掉近距离头像构图，让它小比例进入具体工作场景。
- 角色不像：局部编辑并强化黑色身体、大白眼、小黑瞳、三角耳、少量乱线胡须。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量图应该先让读者看懂一个清爽的低科技隐喻，然后发现里面有一只呆头黑猫正在认真搞错或搞定某个关键动作。

如果第一眼像黑猫肖像、宠物贴纸、粗糙水彩画或教程页，而不是白纸上的怪诞产品草图，就不合格。
