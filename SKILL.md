# Ian 风呆头黑猫正文配图

## 核心定位

为中文文章设计和生成 16:9 横版正文配图。目标不是做精致商业插画、PPT 信息图、可爱宠物头像或粗糙涂鸦海报，而是把文章里的关键判断、流程、结构、状态或隐喻，变成一张接近 Ian 原版小黑项目的清爽手绘解释图。

默认视觉 IP 是“呆头黑猫”：黑色小水彩平涂身体，大白眼，黑色小瞳孔，三角耳，少量乱线胡须，呆住的脸，像一只误入复杂系统却认真捣鼓的 Q 版黑猫。黑猫必须参与画面的核心动作，不能只是趴在角落当装饰。

整体画面必须优先继承原版小黑项目的风格：纯白背景、细黑手绘线、轻微抖动、大量留白、少量红橙蓝中文批注、低科技隐喻、结构清楚但不说明书。黑猫只是替代小黑的固定 IP，不要让整张图变成粗糙水彩画或杂乱涂鸦。

黑猫在画面中不要太大：默认只占画布约 8%-18%，承担动作即可；需要强调角色状态时最多约 25%。除非用户明确要求头像或封面，不要使用近距离大头猫作为主画面。

按任务需要读取，不要一次塞满上下文：

## 工作流

### 1. 消化正文

先读用户给的正文、链接、Notion 页面、Markdown 文件或截图内容。提炼：

- 核心观点是什么
- 哪些段落承担认知转折
- 哪些内容适合用图解释
- 哪些地方只适合文字，不需要图

不要平均配图。优先选择“认知锚点”，例如：核心判断、两个断点、输入输出闭环、分流、前后对比、一物多用、承接路径、常见坑、角色状态变化。

### 2. 先出配图策略

如果用户只是说“分析怎么配图 / 思考哪些地方需要配图”，先给 shot list。每张图写清楚：

- 放在哪个段落后
- 图的主题
- 核心意思
- 结构类型
- 呆头黑猫在图里做什么
- 黑猫的姿势、表情、动作占比
- 整体场景如何保持 Ian 原版清爽风格
- 建议元素
- 建议中文标注词

默认 4-8 张。文章很短时 1-3 张；长文也不要轻易超过 9 张。够用就好，避免把正文做成画册。

### 3. 单张生成

如果用户明确要求“生成 / 输出 / 做图 / 帮我生成”，不要停下来等确认；用内置 `image_gen` 每张单独生成。不要把多张图拼在一张里。

每张图只讲一个核心结构。提示词必须包含：

- 16:9 横版中文正文配图
- 纯白背景
- 黑色细手绘线稿，轻微抖动，整体干净清爽
- 小比例呆头黑猫作为核心动作主体
- 大白眼、小黑瞳、三角耳、乱线胡须、呆滞表情等固定识别点
- 黑猫占画布约 8%-18%，最多 25%，不抢结构主体
- 少量中文手写批注，最多 5-8 处，每处 2-8 个字
- 可有轻微水彩毛边，只限黑猫身体，不扩散成背景纹理
- 禁止 PPT、商业矢量、粗糙满屏涂鸦、真实猫、写实水彩、儿童绘本甜美风、左上角类型标题

不要复刻参考图或过往案例。参考图只提供角色识别点，不能直接复用“大头近景 / 只露脸 / 直视镜头头像构图”。除非用户明确要头像，否则默认让黑猫以小比例全身或半身进入结构场景，做出抓、扒、推、趴、钻、缠线、叼纸卡等动作。

### 4. 检查与迭代

生成后检查 `references/qa-checklist.md`。如果出现以下问题，优先重生成或局部编辑：

- 黑猫只是装饰
- 黑猫仍是近距离头像，没有动作
- 黑猫太大，抢走结构主体
- 角色不像固定 IP
- 画面太粗糙、太脏、太水彩、太涂鸦
- 画面太像流程图/PPT
- 中文太多或错字严重
- 左上角出现“流程图/系统架构图/路线图”等标题
- 画风太可爱、太甜、太儿童绘本或太写实
- 背景不是干净白底

### 5. 保存交付

如果用户在 workspace 内工作，把最终图复制到：

```text
assets/<article-slug>-illustrations/
```

按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

保留原始生成文件，不要覆盖已有资产，除非用户明确要求替换。

## 输出口径

生成前的策略输出要短而准。生成后的交付要包含：

- 生成了几张
- 每张图的用途
- 保存路径
- 哪些图最稳，哪些图是可选

不要长篇解释风格理论；让图自己说话。
